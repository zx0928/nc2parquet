## Key Patterns

- **Flat combination buffer**: `CombinationBuffer` in `src/extract.rs`; packs dimension index combinations as `Vec<usize>` with stride indexing; avoids per-row heap allocation
- **Bounding-box slab read**: `extract_data_batch` uses `var.get_values::<f32, _>(extents)` once per variable; use `netcdf::Extents::try_from((starts.as_slice(), counts.as_slice()))` for multi-dimensional extents
- **Eager drop via inner scope**: `let result = { let temp = ...; result };` frees `coordinate_vars` and `slab` before `build_dataframe` allocates Polars columns
- **Additive output config threading**: `build_parquet_writer` in `src/output.rs` accepts `Option<&OutputConfig>`; `None` produces a default writer
- **Binary-library split**: `handlers/` declared in `main.rs` not `lib.rs`; CLI-specific deps (indicatif, anyhow) stay out of lib
- **Error enum with boxed SDK variant**: `StorageError` is `Box<StorageError>` inside `Nc2ParquetError`; requires manual `From` impl; `#[from]` does not support boxed variants
- **Processor lazy batching**: `ProcessingPipeline::execute` batches consecutive `UnitConverter` processors into a single `.with_columns(batch_exprs).collect()` call
- **Include-based CI matrix**: use `matrix.include` entries (not Cartesian product) to control OS/toolchain pairs precisely; `fail-fast: false` for maximum diagnostic visibility
- **Separate cache key suffix per job**: test job uses `-cargo-`, coverage uses `-cargo-cov-`, benchmark uses `-bench-`; prevents instrumented artifacts from polluting incremental compilation
- **`taiki-e/install-action` for pre-built Cargo tools**: eliminates 2-3 min compile time for tools like `cargo-tarpaulin` that have pre-built releases
- **cargo-dist `dist-workspace.toml` as single source of truth**: all target triples and release config go in `dist-workspace.toml`; `release.yml` is regenerated by `cargo dist init`; never hand-edit `release.yml`
- **Soft benchmark alerting**: `fail-on-alert: false` + `comment-on-alert: true` + 150% threshold; avoids flaky CI from runner performance variance (5-10%)
- **`auto-push` conditional on event type**: benchmark baselines pushed to `gh-pages` only on main-push events, not PRs; feature branches cannot corrupt the performance history

## Architecture and API Boundaries

- `src/lib.rs` is the only public surface; `extract.rs` and `output.rs` are `pub(crate)`
- `src/handlers/` must not be imported by `lib.rs` or any library module
- Integration tests belong in `src/tests/` (not `tests/`) because they need `pub(crate)` access to internal types; `tests/` is for black-box external tests
- `PostProcessor` trait and `NCFilter` trait are the two primary extension points; new processors/filters require: trait impl, serde config struct, factory wiring, integration test
- `ProcessorConfig` uses `#[serde(tag = "type", rename_all = "snake_case")]`; `CompressionCodec` uses `#[serde(rename_all = "lowercase")]`
- `StorageError` variants stay inside `StorageError`, not promoted into `Nc2ParquetError`

## Known Limitations and Technical Debt

- Doc test count: 28 (down from 39 after epic-04); root cause not confirmed; investigate with `cargo test --doc -- --list` before adding more doc examples
- `FormulaApplier` does not support unary minus prefix (`-temperature`); write `0 - temperature` instead — see `src/postprocess.rs` line 978
- `FormulaApplier` and `DateTimeConverter` do not implement `to_lazy_expr()`; each materializes a DataFrame per call
- `Polars 0.51.0`: `AggregationOp::Mean` panics in release builds via streaming engine; `AggregationOp::Sum` works
- `BatchConfig` does not support multi-variable extraction (uses `variable_name: String`)
- Multi-variable extraction opens the first `netcdf::Variable` three times due to borrow checker constraints — see `src/extract.rs` lines 395-428
- Homebrew cache not persisted between macOS CI runs; `brew install netcdf` adds 30-60 sec per run
- Benchmark baseline must be seeded by a first push to main after `benchmark.yml` is merged; first run has no baseline to compare against
